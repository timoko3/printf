# Учебная работа "Printf" 

## Описание 
В рамках данного проекта реализована собственная версия функции **printf** из стандартной библиотеки языка C (**libc**). Реализация выполнена на ассемблере для архитектуры x86-64 в операционной системе linux ubuntu.

Функция принимает форматную строку, содержащую обычный текст и спецификаторы формата. Обычный текст выводится без изменений, а спецификаторы заменяются соответствующими аргументами, переданными в функцию, в соответствии с правилами форматирования.

Поддерживаются следующие спецификаторы:
- **%s** - строка
- **%c** - символ 
- **%d** - целое число 
- **%f** - число с плавающей точкой 
- **%o** - восьмеричное представление 
- **%x** - шестнадцатеричное представление

Дополнительно реализован расширенный спецификатор:
- **%b** -двоичное представление

## Реализация
Первым делом при получение строки моя программа анализирует ее на спецификаторы формата и токенизирует на части которые можно обработать по понятному правилу(спецификаторы и куски обычного текста). В специально выделенный буфер в памяти записываются индексы начала и конца каждого токена и затем каждый из них обрабатывается отдельно. 

Была создана начало каждого токена анализируется на то спецификатор ли это. Если это строка, то она просто копируется в буфер, который в дальнейшем будет выводиться на экран. Основные сложности очевидно заключаются в обработке спецификаторов. 

Для их обработки была создана "джамп-таблица", которая представляет из себя массив из 256 байт состоящий из указателей на функции обработчики спецификаторов или функцию объявляющую пользователю о некорректности введенного спецификатора. То есть обработка спецификатора начинается с прыжка по адресу который, лежит в "джамп-таблице" по адресу соответствующему ASCII коду символа стоящего после '%' в спецификаторе.
